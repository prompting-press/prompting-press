## T005 — Normalizers (`src/error.rs`)

### `From<garde::Report> for ConsumerError`

garde 0.23 has **no `flatten()`**; mapping is via `iter()` over `&(Path, Error)`:

```rust
impl From<garde::Report> for ConsumerError {
    fn from(report: garde::Report) -> Self {
        let rows = report
            .iter()
            .map(|(path, error)| FieldError {
                field: path.to_string(),              // Path: Display -> dot-path
                code: code::VALIDATION.to_string(),   // garde has no machine code (D3)
                message: error.message().to_string(), // Error::message() -> &str
            })
            .collect();
        Self::Validation(rows)
    }
}
```

### `From<KernelError> for ConsumerError` (exhaustive, no wildcard)

```rust
impl From<KernelError> for ConsumerError {
    fn from(err: KernelError) -> Self {
        let row = match err {
            KernelError::UnknownVariant { requested } => FieldError {
                field: "variant".to_string(),
                code: code::UNKNOWN_VARIANT.to_string(),
                message: format!("unknown variant: `{requested}`"),
            },
            KernelError::UndefinedVariable { name } => FieldError {
                field: name.clone(),
                code: code::UNDEFINED_VARIABLE.to_string(),
                message: format!("undefined variable at render: `{name}`"),
            },
            // SEC-004: detail may embed bound-value content — DO NOT copy it.
            KernelError::Parse { detail: _ } => FieldError {
                field: "template".to_string(),
                code: code::PARSE.to_string(),
                message: "template parse error".to_string(),
            },
            // SEC-004: detail may embed bound-value content — DO NOT copy it.
            KernelError::Render { detail: _ } => FieldError {
                field: "template".to_string(),
                code: code::RENDER.to_string(),
                message: "render error".to_string(),
            },
            KernelError::ExcludedFeature { detail: _ } => FieldError {
                field: "template".to_string(),
                code: code::EXCLUDED_FEATURE.to_string(),
                message: "template uses an excluded feature".to_string(),
            },
        };
        Self::Kernel(vec![row])
    }
}
```

### SEC-004 scrub (FR-015) — how it works

The kernel's own `error.rs` doc-comment confirms `Parse` / `Render` `detail` "may embed
bound-value content (which can be untrusted / PII)". The scrub strategy is **discard, not
truncate**: the `Parse` and `Render` arms bind `detail: _` and emit a **fixed templated
message** (`"template parse error"` / `"render error"`). The raw `detail` is never read, so
no bound value — secret or otherwise — can reach `FieldError::message` or the `Display`
string a log line derives from. The failure class is still recoverable by the consumer via
the stable `code` (`"parse"` / `"render"`).

`UnknownVariant.requested` and `UndefinedVariable.name` are surfaced because they are a
caller-supplied variant identifier and a template variable *name* respectively — neither is
bound *value* content. `ExcludedFeature.detail` names a template construct (a tag), not a
value, but is templated for uniformity / defense in depth.

Two `#[cfg(test)]` SEC-004 tests assert a secret embedded in `Render`/`Parse` detail is
absent from both the structured `FieldError::message` and the `ConsumerError` `Display`
string. Two further tests assert `UnknownVariant`/`UndefinedVariable` surface their
identifier with the stable code.

## T006 — Registry (`src/registry.rs`)

```rust
#[derive(Debug, Clone, Default)]
pub struct Registry {
    prompts: BTreeMap<String, PromptDefinition>, // BTreeMap -> deterministic check() order
}

impl Registry {
    pub fn new() -> Self { Self::default() }
    pub fn insert(&mut self, def: PromptDefinition) {
        let key = def.name.to_string(); // PromptDefinitionName derefs to String
        self.prompts.insert(key, def);
    }
    pub fn get(&self, name: &str) -> Option<&PromptDefinition> { self.prompts.get(name) }
}
```

Loaders (`load_yaml` / `load_json`) deliberately omitted — they belong to US2 (T013). Three
unit tests cover insert→get round-trip, same-name replacement, and empty-registry lookup.

## Summary

- **Builds clean**, **clippy clean** with `-D warnings`, **fmt** applied.
- **SEC-004 scrub tests pass** (2 scrub + 2 surface tests; 7 tests total, all green). Scrub
  is by *discard* — `Parse`/`Render` `detail` is bound `_` and never read; a fixed message
  is emitted, so secrets cannot leak via `message` or `Display`.
- **No `garde::Report` / `KernelError` on any public signature.** Both are consumed
  by-value only inside the `From` impls; `ConsumerError` (closed enum) is the sole public
  error type. The `From<KernelError>` match is exhaustive with no wildcard — a new kernel
  variant becomes a compile error here until mapped.
- `render` / `check` / `compose` modules intentionally not declared (files absent; later
  phases).
